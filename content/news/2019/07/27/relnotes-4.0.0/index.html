Release Notes v4.0.0

27 July 2019

Release Notes v4.0.0

See the full list of issues resolved in this release.

Here are the release highlights:

  • RYA-446 - RPM for installing Query Manager as a daemon process on CentOS 7.

  • RYA-448 - Implement query manager xml configuration marshaling code

  • RYA-449 - Create interfaces for QueryChangeLogSource

  • RYA-450 - Create a KafkaQueryChangeLogSource

  • RYA-451 - Create QueryManager

  • RYA-453 - Implement a Query Manager driver

  • RYA-455 - QueryExecutor should be able to stop all queries of a specific Rya Instance

  • RYA-456 - Implement a Single Node implementation of QueryExecutor

  • RYA-458 - Only be able to configure a single Kafka based QueryChangeLogSource

  • RYA-462 - Update query result topic name for Rya Streams

  • RYA-463 - Add insert flag to StreamsQuery

  • RYA-467 - Rya Streams - QueryChangeLog topic deletes content after 7 days.

  • RYA-469 - Rya Streams - Join iterator bug.

  • RYA-485 - fix maven-shade-plugin usage

  • RYA-488 - The Geo dependency repositories need to be in a geoindexing profile.

  • RYA-492 - Add language support to Literals

  • RYA-494 - Shell insert and query bug.

  • RYA-495 - old version numbers in Vagrantfile prevent it from working

  • RYA-497 - Accumulo Rya Kafka Connect Sink - Slow insert rates.

  • RYA-487 - Kafka Connect Rya Sink

  • RYA-515 - Upgrade Mongo driver to 3.10.2

  • RYA-405 - Migrate from Sesame to rdf4j libs

  • RYA-466 - Rya Streams Client - Stream results to file.

  • RYA-489 - Create the rya.test.accumulo project.

  • RYA-491 - Create the rya.test.mongo project.

  • RYA-269 - Test AccumuloLoadStatementsFileIT fails: depends on old identifier: “urn:mvm.rya/2012/05#version”

  • RYA-476 - DateTimeWithinPeriodTest is sensitive to DST spring forward.

  • RYA-443 - Implement a single node query manager

  • RYA-486 - Update master to 4.0.0
